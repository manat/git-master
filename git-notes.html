## Why Git? 1. Distributed 2. Fast 3. Safe. ## Snapshots ![Normal VCS](http://git-scm.com/figures/18333fig0104-tn.png "Normal VCS") ![Git](http://git-scm.com/figures/18333fig0105-tn.png "Git") After all, git tracks contents and not filenames. When you add a file to your repo, the filename is in the tree object. The actual content of the file gets added as a blob object in your repo. If you add another file of the same content, git will not add another blob for it. See [The Git Object Model](http://schacon.github.io/gitbook/1_the_git_object_model.html) for more information. ## States ![Git States](http://graph1zzlle.github.io/images/posts/git-commit-workflow.png "Git States") ### Diff ``` git diff git diff HEAD^ git diff --staged git diff HEAD^ myfile git difftool ``` ## Walkthrough * [Git Immersion](file:///C:/Users/dt77850/Documents/Git/tutorials/gitimmersion/git_tutorial/html/lab_01.html) * [Interactive Git](http://pcottle.github.io/learnGitBranching/?NODEMO) Starts Git basic with Git Immersion (1 - 25). #### Add - Interactive Staging ``git add -i`` See [Git Tools - Interactive Staging](http://git-scm.com/book/mk/Git-Tools-Interactive-Staging) Then continue with Moving Work Around on learnGitBranching. ### cherry-pick ``` git checkout another_branch #or checkout HEAD~3 (ancestors) git cherry-pick d6982b2 c68d52b 20b7e24 ``` A real life use case of cherry-pick is that we can choose to pick specific commits from the different branch instead of merging all of them. ## Branch _Branches as Silos_ ![branches as silos](http://git-scm.com/figures/18333fig0319-tn.png "branches as silos") _Topic Branches_ ![topic branches](http://git-scm.com/figures/18333fig0320-tn.png "topic branches") Now, let's say you decide you like the second solution to your issue best (iss91v2); and you showed the dumbidea branch to your coworkers, and it turns out to be genius.
